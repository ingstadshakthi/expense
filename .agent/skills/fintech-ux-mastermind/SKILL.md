# Fintech & Analysis Mastermind Skill

When activated, you must evaluate the expense tracker features against these core fintech principles:

### 1. The "Glanceability" Test

- **Mastermind Principle:** Users shouldn't "study" a dashboard; they should "sense" it.
- **Audit:** Are spending categories color-coded consistently? Is the "Total Balance" or "Monthly Spend" the largest visual element? Use $size$ and $weight$ to define importance.

### 2. Analysis & Actionable Insights

- **Mastermind Principle:** Data without a "Next Step" is just clutter.
- **Audit:** For every chart (Pie/Bar), suggest a text-based "Mastermind Insight" (e.g., "You've spent 15% more on Dining than last month. Want to set a limit?").

### 3. Frictionless Logging

- **Mastermind Principle:** The harder it is to add an expense, the less the app is used.
- **Audit:** Review the "Add Expense" flow. Can it be done in 3 taps/clicks or less? Suggest AI-auto-categorization to reduce manual typing.

### 4. Trust & Security Cues

- **Mastermind Principle:** Money is emotional. The UI must feel "solid."
- **Audit:** Check for "soft" elements—rounded corners for friendliness, but precise typography for numbers (monospaced fonts for digits to prevent jumping).

### Output Format

1. **Critical Friction Points:** What is stopping the user from tracking today?
2. **Analysis Upgrade:** How to make the AI insights feel more "human" and helpful.
3. **Visual Polish:** CSS/Component tweaks for a premium "Mastermind" look.
